# Slate

**Claude Code에 카드뉴스·캐러셀·피겨 편집기를 더해주는 오픈소스 Skill입니다.**

Slate는 디자인이 담긴 폴더를 Claude Code 미리보기 패널 안의 Figma 스타일 라이브
캔버스로 열어줍니다. 사용자는 시각적으로 배치하고 편집하고, Claude는 같은 파일을
편집하며, 그 변화가 실시간으로 화면에 반영됩니다. 도움이 필요할 때는 요소를 선택해
요청하면, Claude 서브에이전트가 그 부분을 수정하는 동안 사용자는 계속 작업할 수
있습니다.

디자인 전체가 하나의 JSON 파일이라 어떤 것도 잠겨 있지 않습니다. 캔버스에 보이는
그대로 PNG·SVG·PDF로 내보내집니다.

## 주요 기능

- **양방향 실시간 동기화.** 덱 전체가 하나의 `cards.json`입니다. 캔버스에서
  드래그하고 입력하면, Claude가 파일을 편집하고, 둘이 항상 동기화됩니다. 채팅으로
  작업하든 보드에서 Claude에게 지시하든 같은 파일, 같은 결과입니다.
- **태그하고 요청하기.** 요소나 페이지를 선택하면 컴포저에 칩으로 붙습니다. 수정을
  요청하면 서브에이전트가 정확히 그 소재만 처리합니다. 요청을 여러 번 보내면
  병렬로 실행되며, 각 에이전트는 자기 실행 셸 로그를 실시간으로 보여줍니다.
- **라이브 커서.** 모든 편집이 diff되어 이름표가 달린 커서로 캔버스 위를 움직이며
  표시됩니다. 여러 에이전트가 동시에 작업하는 모습도 볼 수 있습니다.
- **팬시가 기본.** 글래스모피즘 스타일 엔진(그라디언트·블러·드롭섀도·투명도)을
  순수 SVG로 렌더링하므로, 내보낸 결과가 화면과 정확히 일치합니다.
- **14종 디자인 시스템 내장.** glass · apple · linear · vercel · stripe · figma ·
  raycast · notion · spotify · nike · theverge · framer · cal · slate. 하나를
  고르면 모든 편집이 그 시스템을 따릅니다. (12종은
  [awesome-design-md](https://github.com/VoltAgent/awesome-design-md), MIT)
- **제대로 된 에디터.** 모든 슬라이드를 한 Figma 스타일 보드에, 드래그로 순서를
  바꾸는 레이어 패널, 다중 선택·마퀴, 스마트 정렬 가이드, 회전, 줌, 드래그로 선긋기,
  이미지 드롭, 언두/리두, 복원 가능한 버전 히스토리.
- **깔끔한 내보내기.** PNG·SVG·PDF — 하나의 직렬화기를 공유하므로 보이는 그대로
  나옵니다.

## 시작하기

```bash
git clone https://github.com/AnYejun/slate.git ~/.claude/skills/slate
```

Claude Code를 열고 시각적인 작업을 요청하세요 — 예: *"카드뉴스 5장 만들어줘"*,
*"논문에 넣을 피겨 하나 만들어줘"*. Claude가 워크스페이스를 준비하고 미리보기
패널(또는 `localhost:5173` 전체화면)에 에디터를 열어, 함께 편집하게 됩니다.

**요구사항.** Node 18+. 보드에서 서브에이전트를 실행하려면
[Claude Code CLI](https://claude.com/claude-code)에 한 번 로그인해 두세요
(`claude` → `/login`). 실행은 별도 API 키가 아니라 Claude 구독으로 처리됩니다.
CLI가 없으면 보드가 요청을 복사해 주어 채팅에 붙여넣어 쓸 수 있습니다.

## 사용법

### 캔버스 이동 (Figma 방식)

모든 슬라이드가 하나의 보드에 나란히 놓입니다. 익숙한 방식으로 탐색하세요:

| 동작 | 방법 |
|---|---|
| **팬(이동)** | **Space**를 누른 채 드래그, 또는 **가운데 마우스 버튼** 드래그 |
| **스크롤** | 트랙패드 두 손가락 스크롤; **Shift + 스크롤**은 가로 이동 |
| **줌 인/아웃** | **⌘ / Ctrl + 스크롤**, 또는 상단바의 **− / +** 버튼 |
| **화면 맞춤** | 상단바의 **줌 %** 클릭(맞춤으로 초기화) |
| **슬라이드로 이동** | 왼쪽 **Pages** 스트립의 썸네일 클릭 |

Space를 누르는 동안 커서가 손 모양으로 바뀌어, 드래그하면 선택 대신 캔버스가
움직입니다 — Figma와 동일합니다.

### 선택과 편집

- **선택** — 캔버스에서 요소를 클릭하거나 **Layers** 목록의 행을 클릭합니다.
  오른쪽 **Design** 패널에 속성이 표시됩니다.
- **다중 선택** — **Shift-클릭**으로 요소를 추가하거나, 빈 캔버스에서
  **마퀴 드래그**로 범위 선택합니다. 패널이 정렬/분배 컨트롤로 바뀝니다.
- **이동** — 선택한 요소를 드래그(선택된 것들이 함께 이동). 방향키는 1px,
  **Shift + 방향키**는 10px 이동.
- **크기 조절** — 사각 핸들 드래그. **회전** — 선택 위의 원형 핸들 드래그
  (**Shift**는 15° 스냅).
- **텍스트 편집** — 텍스트 요소를 더블클릭해 입력. 한글 IME 처리됨.
- **스마트 가이드** — 드래그 시 다른 요소의 모서리/중심과 카드 중심에 스냅되며
  정렬선이 표시됩니다.
- **삭제** — Delete / Backspace. **실행 취소/다시 실행** — ⌘Z / ⌘⇧Z.

### 툴바 (상단 가운데)

Heading · Text · Rectangle · Ellipse · **Line** · **Image**.

- **Line(선)** — 도구를 클릭한 뒤 **캔버스에서 드래그**해 임의 각도로 그립니다.
- **Image(이미지)** — 클릭해 파일을 고르거나, **이미지를 캔버스에 그냥 드롭**하세요.
  data URI로 임베드되어 내보내기가 자체 완결됩니다.

### Pages & Layers (왼쪽 패널)

- **Pages** 스트립 — 모든 슬라이드의 썸네일; **+**로 추가; 호버해서 복제/삭제.
  클릭하면 해당 슬라이드가 활성화됩니다.
- **Layers** 목록 — 활성 슬라이드의 모든 요소(최상위 먼저). 행을 **드래그**하면
  쌓임 순서가 바뀝니다. 클릭 선택, Shift-클릭 다중 선택.

### Claude와 함께 작업하기 (Agents 탭)

Slate의 핵심 아이디어입니다: **바꾸고 싶은 것을 가리키고, 요청하세요.**

1. **소재 태그.** 캔버스에서 요소를 하나 이상 선택하거나(또는 Pages 스트립으로
   페이지 전체) 선택합니다. **Agents** 탭을 열면 선택이 컴포저에 칩으로
   표시됩니다(예: *"2 elements · Cover"*). 칩을 지우면 덱 전체를 대상으로 대화합니다.
2. **한 가지 지시.** 입력하고 **Enter**(줄바꿈은 Shift+Enter). 예: *"더 대담하게"*,
   *"글래스 스타일로 다시"*, *"이것들 정렬하고 간격 좁혀줘"*.
3. **실행을 지켜보기.** 진짜 헤드리스 `claude -p` 서브에이전트가 태그한 소재에만
   범위를 잡고 시작합니다. 그 **실행 셸**이 탭에 스트리밍되고(어떤 파일을 읽고 무엇을
   바꾸는지), 편집하는 동안 **이름표가 달린 커서**가 캔버스 위를 움직입니다.
   **follow**(눈 아이콘)를 켜면 작업 중인 곳으로 자동 스크롤됩니다.
4. **여러 개 동시에.** 다른 소재를 태그하고 또 요청하면 작업이 **병렬로** 실행되며,
   각각 자기 카드·셸·커서를 가집니다. 카드의 **✕**로 해당 에이전트를 중지합니다.

태그 없이 덱 전체 명령을 내리거나, 그냥 일반 채팅창에서 Claude에게 요청해도 됩니다 —
Slate는 평범한 Claude Code 스킬이라 둘 다 같은 파일에서 동작합니다.

> **에이전트를 실행하려면** Claude Code CLI에 한 번 로그인하세요(`claude` → `/login`).
> 실행은 Claude 구독으로 처리됩니다. 로그인이 없으면 보드가 요청을 클립보드에
> 복사해 주어 채팅에 붙여넣어 쓸 수 있습니다.

### 스타일 & 효과

- **디자인 시스템** — Design 패널의 **Style** 드롭다운에서 내장 14종(glass, apple,
  linear, vercel, stripe, figma, …) 중 선택합니다. 선택은 `design.md`가 되어 모든
  에이전트가 읽고 따릅니다.
- **배경·채움**은 그라디언트를 받습니다: `linear:#0B1020,#1B1140,160`
  (시작, 끝, 각도).
- **요소별 효과**(인스펙터): **Opacity**, **Blur**(글로우/헤일로), **Shadow**
  (`dx dy blur color`) — 전부 순수 SVG라 그대로 내보내집니다.

### 내보내기

우상단 **Export**: **PNG**(현재 슬라이드, 2×), **SVG**(현재 슬라이드, 벡터),
**PDF**(전체 슬라이드). 캔버스 렌더러와 내보내기가 같은 코드라 파일이 화면과 정확히
일치합니다.

### 버전 히스토리

**시계** 아이콘으로 히스토리를 엽니다: 변경마다 자동 스냅샷 + 이름 붙인 체크포인트,
각각 한 번의 클릭으로 복원 가능. 에이전트의 모든 편집도 먼저 스냅샷되므로 잃어버릴
일이 없습니다.

## 동작 방식

```
요소 태그 ──► POST /api/agents/run ──► 요청마다 claude -p
    ▲                                   (Read / Edit / Write, 샌드박스)
    │                                            │
라이브 캔버스 ◄── 파일 diff → 이름표 커서 ◄── 에이전트가 cards.json 편집
(React + Vite)     셸 출력 → 워크보드                └─ 자동 스냅샷, 복원 가능
```

`cards.json`이 단일 진실원본입니다. 개발 서버가 이 파일을 감시하며 변경마다
diff하고, 그 변경을 수행한 에이전트에 귀속시키고, 히스토리용으로 스냅샷을 남기고,
커서와 셸 출력을 보드로 스트리밍합니다. 화면 SVG 직렬화기가 곧 내보내기 엔진이라
캔버스와 결과물이 동일합니다.

## 참고

- **안전성.** 서브에이전트는 `Read/Edit/Write`만, 워크스페이스 디렉터리 안에서,
  타임아웃과 격리된 환경으로 실행됩니다. 외부 편집은 항상 먼저 스냅샷되며 어떤
  버전이든 한 번의 클릭으로 복원할 수 있습니다.
- **채팅도 그대로.** Slate는 평범한 Claude Code 스킬이라 언제든 채팅으로 Claude에게
  요청할 수 있습니다. 보드는 설명보다 가리키는 편이 쉬울 때를 위한 것입니다.
- **한글 입력.** IME 조합을 처리하므로 조합 중 Enter로 잘못 전송되지 않습니다.
